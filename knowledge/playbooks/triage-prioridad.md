# Triage de prioridad

Sophia debe calcular prioridad con criterio operativo antes de preparar tickets cuando el usuario no indique prioridad o impacto suficiente.

## Preguntas de severidad

Preguntar de forma breve:

- ¿Afecta solo a una persona, a varios usuarios o a un área completa?
- ¿Bloquea la operación o permite seguir trabajando parcialmente?
- ¿Impacta ventas, despacho, producción, facturación, caja, bodega u otra operación crítica?
- ¿Desde cuándo ocurre?

No convertir la conversación en formulario. Si el usuario responde parcialmente, usar lo disponible. Si dice `crear de todos modos`, continuar con la solicitud.

## Reglas de prioridad sugerida

Usar `Alta` cuando:

- afecta a varios usuarios, un departamento, una planta o un área completa;
- bloquea la operación;
- impide ventas, despacho, producción, facturación, caja o bodega;
- el usuario indica explícitamente urgencia, criticidad o prioridad alta.

Usar `Media` cuando:

- afecta a un usuario;
- permite continuar trabajando parcialmente;
- no hay impacto crítico confirmado.

Usar `Baja` solo cuando:

- el usuario lo solicita explícitamente;
- es una solicitud planificada o mejora sin urgencia operativa.

## Redacción recomendada

Sophia debe explicar que las preguntas sirven para asignar la prioridad correcta, no para retrasar el ticket.
Ejemplo: `Antes de prepararlo, necesito calcular mejor la prioridad para que soporte lo atienda con el nivel correcto.`
